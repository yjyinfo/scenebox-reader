## 📦 SceneBox 소개

**SceneBox Reader**는 **[SceneBox App](https://apps.apple.com/app/id6789491378)**에서 내보낸 보고서를 읽기 위한 전용 리더입니다.

> 현장에서 무슨 일이 있었나요? 문제는 정확히 어디에 있나요? 어떤 증거가 있었나요?

SceneBox는 이러한 답을 완벽하게 기록하고, 누구나 한눈에 이해할 수 있는 보고서로 만들어줍니다.

설치 작업, AS 수리, 현물 인수인계, 시설 점검, 설비 유지보수, 프로젝트 검수, 민원 처리 등 현장 기록을 명확하게 남겨야 하는 모든 순간에 활용할 수 있습니다.

### SceneBox가 하는 일

**🎯 한 번 촬영, 전체 파악** — 텍스트, 사진, 동영상, 음성, 위치를 한곳에 모으세요. 중요한 정보가 앨범, 메모 앱, 채팅방에 흩어지는 일을 멈추세요. 내용이 명확해질 때까지 언제든 항목을 추가하거나 재정렬할 수 있습니다.

**✏️ 문제를 직접 주석 처리** — 사진 위에 화살표와 텍스트를 추가해 결함, 손상, 설치 위치 등 주의가 필요한 부분을 표시하세요. 주석은 이동, 크기 조절, 회전, 색상 변경이 가능합니다. 원본 사진은 항상 보존됩니다.

**⏱️ 정확한 순간을 핀** — 이상한 소리가 들리거나 오디오/비디오에서 핵심 프레임을 발견하면 해당 타임스탬프에 이벤트를 추가하세요. 나중에 검토할 때 해당 순간으로 바로 이동합니다 — 처음부터 다시 찾을 필요가 없습니다.

**📶 오프라인으로 촬영, 계정 불필요** — 가입할 필요 없습니다. 지하실, 서버실, 건설 현장, 통신이 끊기는 어디서든 계속 촬영, 녹음, 정리, 미리보기가 가능합니다.

SceneBox는 이러한 보고서를 표준 `.epub` 파일로 내보내며, **SceneBox Reader**는 어떤 기기에서든 설치 없이 가장 쉽게 읽을 수 있는 방법입니다.

> 💡 **SceneBox Reader는 완전히 표준을 준수하는 EPUB 리더이기도 합니다** — Project Gutenberg, Standard Ebooks 또는 다른 어느 곳의 `.epub` 책이든 열 수 있습니다.

## ✨ 기능

- **🔒 100% 클라이언트 사이드** — 파일은 `JSZip`으로 로컬에서 파싱되며, 어떤 서버에도 업로드되지 않습니다.
- **📐 세 가지 보기 모드** — 단일 페이지 연속 / 두 페이지 / 두 페이지 연속.
- **🔍 스마트 확대/축소** — 모바일/태블릿에서 자동 맞춤, 데스크톱에서 100%. `Ctrl/⌘ + 휠`로 재정의 가능.
- **🖨️ 인쇄** — 모든 페이지를 단일 PDF로 합치며, 브라우저 기본 대화상자에서 페이지 범위를 선택할 수 있습니다.
- **🌗 시스템 다크 모드** — Windows/macOS/Linux/iOS/Android에서 OS 테마를 따릅니다. 수동 재정의는 유지됩니다.
- **🌍 5개 언어** — English / 中文 / 日本語 / 한국어 / Español. 브라우저에서 자동 감지되며 전환 가능합니다.
- **📑 목차** — 챕터 탐색이 가능한 접을 수 있는 사이드바.
- **🎬 미디어 인식** — 이벤트 링크가 있는 인라인 `<audio>`/`<video>` (타임스탬프 클릭 → 해당 위치로 이동). 누락된 매니페스트 항목도 작동하도록 에셋 URL이 즉시 blob URL로 재작성됩니다.
- **🖨 EPUB 고정 레이아웃 친화적** — 사전 페이지 매김 FXL EPUB이 적절한 페이지 나눔과 함께 원본 비율로 렌더링됩니다.
- **📱 모바일 반응형 UI** — 폰에서는 컴팩트 도구 모음, 데스크톱에서는 전체 도구 모음.

## 🚀 빠른 시작

```bash
# 요구 사항: Node.js 18.18+ (Next.js 16 요구 사항)
git clone https://github.com/yjyinfo/scenebox-reader.git
cd scenebox-reader
npm install
npm run dev
```

<http://localhost:3000>을 여세요 (3000번 포트가 사용 중이면 Next가 자동으로 3001로 전환합니다), 그리고:

- 빈 드롭 존의 **아무 곳이나 클릭**하거나,
- 페이지에 **아무 `.epub` 파일이나 드래그**하세요.

끝입니다. 백엔드도, 업로드도, 원격 측정도 없습니다.

## 🛠️ 기술 스택

- **[Next.js 16](https://nextjs.org/)** (App Router) + **[React 19](https://react.dev/)**
- **[TypeScript 5](https://www.typescriptlang.org/)**
- **[Tailwind CSS v4](https://tailwindcss.com/)**
- **[JSZip 3](https://stuk.github.io/jszip/)** — 브라우저 내 EPUB 파싱용

EPUB 라이브러리 없음 — 다음을 수행하는 자체 경량 파서(`lib/epub.ts`)를 직접 작성했습니다:
1. OPF 매니페스트를 찾기 위해 `META-INF/container.xml` 읽기
2. 메타데이터 / 스파인 / 목차 파싱
3. 모든 상대 에셋 참조(`<img src>`, `<video src>`, `<a href>`...)를 zip에서 추출한 blob URL로 재작성
4. 인라인 `<script>`에서 `<![CDATA[]]>` 래퍼 제거 (HTML 파서가 이것에 걸려 문제가 됩니다 — 자세한 사연은 `lib/epub.ts` 참조)

전체 커스텀 파싱 코드: **약 200줄**.

## 🌍 i18n

5개 언어가 [`lib/i18n.ts`](lib/i18n.ts)에 있습니다. 새 언어를 추가하려면:

1. 2자리 코드 선택 (예: `fr`)
2. `fr: { ... }` 블록 추가 (`en`에서 복사하고 번역)
3. `LANGS` 배열에 추가
4. `detectLang()`의 접두사 맵에 `"fr"` 추가

PR을 환영합니다 — [기여하기](#-기여하기)를 참조하세요.

## 🖨️ 인쇄 아키텍처

🖨 버튼을 클릭하면:

1. 모든 스파인 XHTML 페이지를 하나의 숨겨진 iframe으로 병합
2. 각 페이지의 `body { padding/margin }` 제거 (화면 전용 여백 시뮬레이션이며, 프린터에는 자체 물리적 여백이 있음)
3. 10mm 안전 영역으로 A4 맞춤 (기계적 인쇄 불가 영역 회피)
4. `iframe.contentWindow.print()` 호출 — 브라우저가 전체 페이지 범위 제어가 가능한 기본 대화상자 표시

브라우저의 인쇄 미리보기(`chrome-untrusted://print/...`)에서 보이는 PDF는 **브라우저가 생성**한 것이지 우리가 생성한 것이 아닙니다. 우리는 iframe의 HTML/CSS만 제어하며, 나머지는 브라우저가 처리합니다.

## 🔒 프라이버시

- 백엔드, 서버, API가 없습니다.
- 파일 바이트는 브라우저의 JSZip에 의해 완전히 처리됩니다.
- 사용자 기본 설정(`theme`, `lang`) 외에 분석, 쿠키, `localStorage`가 없습니다.
- 어떤 정적 호스트(Vercel / Netlify / GitHub Pages / 자체 서버)에서도 자체 호스팅하기에 안전합니다.
