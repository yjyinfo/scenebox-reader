## 📦 SceneBoxについて

**SceneBox Reader**は、**[SceneBox App](https://apps.apple.com/app/id6789491378)**からエクスポートされたレポートを読むための専用リーダーです。

> 現場で何が起きましたか？問題はどこにあるのでしょうか？証拠は何でしたか？

SceneBoxは、これらの答えを完全に記録し、誰もがひと目で理解できるレポートにまとめます。

設置作業、アフター修理、物件引き渡し、設備点検、機器メンテナンス、プロジェクト検収、自治体への通報など、現場での明確な記録が必要なあらゆる場面で活躍します。

### SceneBoxでできること

**🎯 ワンキャプチャで全体像** — テキスト、写真、動画、音声、位置情報を1箇所にまとめます。写真アプリ、メモアプリ、チャットログに重要な情報を散らばせません。エントリーはいつでも追加・並べ替えでき、内容が明確になるまで整理できます。

**✏️ 問題を直接注釈** — 写真に矢印やテキストを追加して、不具合、損傷、設置場所、その他注目すべき点を指し示します。注釈は移動・リサイズ・回転・色変更が可能です。元の写真は常に保持されます。

**⏱️ 正確な瞬間をピン留め** — 異音に気づいたり、音声／動画の重要なフレームを見つけたら、そのタイムスタンプにイベントを追加できます。後からの確認では、その瞬間に直接ジャンプします。最初から探し直す必要はありません。

**📶 オフラインでキャプチャ、アカウント不要** — サインアップは不要です。地下、サーバールーム、建設現場、電波の届かない場所でも、撮影、録音、整理、プレビューを続けられます。

SceneBoxはこれらのレポートを標準的な`.epub`ファイルとしてエクスポートします。**SceneBox Reader**は、どんなデバイスでも、何もインストールせずにそれらを読むための最も簡単な方法です。

> 💡 **SceneBox Readerは完全に標準的なEPUBリーダーでもあります** — Project Gutenberg、Standard Ebooks、その他どこからでも任意の`.epub`ブックを開けます。

## ✨ 機能

- **🔒 100%クライアントサイド** — ファイルは`JSZip`でローカルに解析され、いかなるサーバーにもアップロードされません。
- **📐 3つの表示モード** — 1ページ連続／見開き／見開き連続。
- **🔍 スマートズーム** — モバイル／タブレットでは自動フィット、デスクトップでは100%。`Ctrl/⌘ + ホイール`で上書き。
- **🖨️ 印刷** — 全ページを1つのPDFに結合し、ブラウザ標準のダイアログでページ範囲を選択可能。
- **🌗 システムダークモード** — Windows/macOS/Linux/iOS/AndroidでOSテーマに追従。手動上書きは保持されます。
- **🌍 5言語対応** — English／中文／日本語／한국어／Español。ブラウザから自動検出、切替可能。
- **📑 目次** — 折りたたみ可能なサイドバーで章単位のナビゲーション。
- **🎬 メディア対応** — インラインの`<audio>`／`<video>`とイベントリンク（タイムスタンプをクリック → シーク）。アセットURLはその場でblob URLに書き換えられ、マニフェストにないエントリでも機能します。
- **🖨 EPUB固定レイアウト対応** — Pre-paginatedのFXL EPUBをネイティブのアスペクト比と適切な改ページでレンダリング。
- **📱 モバイル対応UI** — スマートフォンではコンパクトなツールバー、デスクトップではフルツールバー。

## 🚀 クイックスタート

```bash
# 要件: Node.js 18.18+（Next.js 16の要件）
git clone https://github.com/yjyinfo/scenebox-reader.git
cd scenebox-reader
npm install
npm run dev
```

<http://localhost:3000> を開きます（3000番が使用中の場合、Nextは自動的に3001に切り替えます）。その後：

- 空のドロップゾーンの**どこかをクリック**、または
- 任意の`.epub`ファイルをページに**ドラッグ**。

以上です。バックエンドもアップロードもテレメトリもありません。

## ⌨️ キーボードショートカット

| ショートカット | アクション |
|---|---|
| `Ctrl/⌘ + スクロール` | ズームイン／アウト |
| `← / →` | 前／次のページペア（見開きモード時） |
| `100%をクリック` | ズームを自動フィットにリセット |
| `☰` | 目次サイドバーの切替 |

## 🛠️ 技術スタック

- **[Next.js 16](https://nextjs.org/)**（App Router）+ **[React 19](https://react.dev/)**
- **[TypeScript 5](https://www.typescriptlang.org/)**
- **[Tailwind CSS v4](https://tailwindcss.com/)**
- **[JSZip 3](https://stuk.github.io/jszip/)** — ブラウザ内でのEPUB解析用

EPUBライブラリは使用せず、独自の小さなパーサー（`lib/epub.ts`）を書きました。以下を行います：
1. `META-INF/container.xml`を読み込みOPFマニフェストを特定
2. メタデータ／スパイン／目次を解析
3. すべての相対アセット参照（`<img src>`、`<video src>`、`<a href>`...）をzipから取り出したblob URLに書き換え
4. インライン`<script>`から`<![CDATA[]]>`ラッパーを除去（HTMLパーサーがこれらで詰まるため。経緯は`lib/epub.ts`を参照）

独自パースコードの総量：**約200行**。

## 🌍 国際化（i18n）

5言語が[`lib/i18n.ts`](lib/i18n.ts)にあります。新しい言語の追加手順：

1. 2文字コードを選択（例：`fr`）
2. `fr: { ... }`ブロックを追加（`en`からコピーして翻訳）
3. `LANGS`配列に追加
4. `detectLang()`のプレフィックスマップに`"fr"`を追加

PRを歓迎します — [コントリビュート](#-contributing)を参照してください。

## 🖨️ 印刷アーキテクチャ

🖨ボタンをクリックすると：

1. スパイン内の全XHTMLページを1つの非表示iframeに結合
2. 各ページの`body { padding/margin }`を削除（これらは画面表示専用のマージンシミュレーションであり、プリンターには物理マージンがあるため）
3. 10mmのセーフゾーン付きでA4に収まるようコンテンツを縮小（機械的な印刷不可領域を回避）
4. `iframe.contentWindow.print()`を呼び出し — ブラウザがネイティブのダイアログを表示し、ページ範囲を完全に制御可能

ブラウザの印刷プレビュー（`chrome-untrusted://print/...`）に表示されるPDFは、私たちではなく**ブラウザが生成したもの**です。私たちが制御するのはiframeのHTML/CSSだけで、残りはすべてブラウザが処理します。

## 🔒 プライバシー

- バックエンドなし、サーバーなし、APIなし。
- ファイルのバイト列はすべてブラウザのJSZipだけで処理されます。
- アナリティクス、Cookie、`localStorage`は一切なし（ただしユーザー設定の`theme`と`lang`を除く）。
- 任意の静的ホスト（Vercel／Netlify／GitHub Pages／独自サーバー）で安全にセルフホスト可能。
